C++17语言特性

  • 类模板的模板参数自动推导
    现在类的构造函数能用来推导类模板参数了，就像推导函数模板那样。

  • 使用auto声明非类型模板参数
    在涉及到所允许类型的非类型模板参数列表[*]中，遵循 auto 的推导规则，可以从其参数类型中推导出模板参数。

    ※ 例如，你不能使用 double 类型的模板参数，这也会使得 auto 的推导无效。

  • 折叠表达式
    折叠表达式会在二进制运算符上执行一个模板参数包的折叠。

    • 形式为 (…op e) 或 (e op…) 的表达式，其中 op 是折叠运算符，e 是未展开的参数包，这种表达式被称为一元折叠。
    • 形式为 (e1 op…op e2) 的表达式，其中 op 是折叠运算符，这种表达式被称为二元折叠。e1或e2其中一个是未展开参数包，但不能两者都是。

  • auto推导braced-init-list的新规则
    使用统一的初始化语法时，对 auto 推导规则进行改变。以前， auto x{3}; 会被推导为 std::initializer_list ，现在会被推导为 int 。

  • constexpr lambda
    编译期lambda表达式使用 constexpr 。

  • lambda表达式按值捕获本对象
    以前在lambda表达式中捕获本对象只能引用捕获（应该指的是捕获this指针）。一个会造成问题的例子：如果在一个异步回调里要捕获对象，这个回调里要求对象可用，但是可能已经超过该对象的生存周期。现在C++17里使用 *this 会对当前对象做一个拷贝，而使用 this (C++11) 将继续使用引用捕获。

  • 内联变量
    内联说明符 inline 既可以用于变量，又可以用于函数。定义为内联的变量和内联函数具有相同的语义。

    还可以用其来声明和定义一个静态成员变量，这样就不需要在源文件中初始化了。
