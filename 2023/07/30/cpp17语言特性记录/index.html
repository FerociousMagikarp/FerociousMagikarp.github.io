C++17语言特性

类模板的模板参数自动推导

现在类的构造函数能用来推导类模板参数了，就像推导函数模板那样。

使用auto声明非类型模板参数

在涉及到所允许类型的非类型模板参数列表[*]中，遵循 auto 的推导规则，可以从其参数类型中推导出模板参数。

※ 例如，你不能使用 double 类型的模板参数，这也会使得 auto 的推导无效。

折叠表达式

折叠表达式会在二进制运算符上执行一个模板参数包的折叠。

  • 形式为 (…op e) 或 (e op…) 的表达式，其中 op 是折叠运算符，e 是未展开的参数包，这种表达式被称为一元折叠。
  • 形式为 (e1 op…op e2) 的表达式，其中 op 是折叠运算符，这种表达式被称为二元折叠。e1或e2其中一个是未展开参数包，但不能两者都是。

auto推导braced-init-list的新规则

使用统一的初始化语法时，对 auto 推导规则进行改变。以前， auto x{3}; 会被推导为 std::initializer_list ，现在会被推导为 int 。

constexpr lambda

编译期lambda表达式使用 constexpr 。

lambda表达式按值捕获本对象

以前在lambda表达式中捕获本对象只能引用捕获（应该指的是捕获this指针）。一个会造成问题的例子：如果在一个异步回调里要捕获对象，这个回调里要求对象可用，但是可能已经超过该对象的生存周期。现在C++17里使用 *this 会对当前对象做一个拷贝，而使用 this (C++11) 将继续使用引用捕获。

内联变量

内联说明符 inline 既可以用于变量，又可以用于函数。定义为内联的变量和内联函数具有相同的语义。

还可以用其来声明和定义一个静态成员变量，这样就不需要在源文件中初始化了。

结构化绑定

一个去结构化初始化的提议，允许写 auto [x, y, z] = expr; 这种表达式，其中 expr 是一个元组形式的对象，其中的元素会被绑定到变量 x, y, z（这个表达式中就声明了）。元组形式包括 std::tuple、std::pair、std::array 以及聚合结构。

带有初始值的选择语句

新版本的 if 和 switch 语句简化了常见的代码结构，并帮助用户使代码更紧凑。

UTF-8字符字面量

以 u8 开头的 char 类型字面量。一个 UTF-8 字面量的值与 ISO 10646 所对应的值相同。

枚举用列表直接初始化

现在枚举能用大括号表达式初始化了。

[[fallthrough]], [[nodiscard]], [[maybe_unused]] attributes

C++17 引入了三种新的attributes：[[fallthrough]]，[[nodiscard]] 以及 [[maybe_unused]]。

  • [[fallthrough]] 告诉编译器在switch语句中不break直接往后执行是预期行为。这个attribute必须在switch语句中使用，并且必须在下一个 case/default 标签前面。
  • [[nodiscard]] 当函数或类含有此attribute，并且返回值被丢弃时，会触发警告。
  • [[maybe_unused]] 告诉编译器某个变量或参数未使用是预期行为。

__has_include

__has_include(operand) 表达式会用在 #if 和 #elif 表达式中检测头文件或源文件 operand 是否可以被包含。

这样做的一个用例是：使用两个以相同方式工作的库，如果系统找不到首选库，就使用另一个备份库。

它还可以用在在不同平台上以不同名称或位置存在的头文件，不知道程序会在哪个平台上运行。OpenGL就是个很好的例子，在macOS上路径为 OpenGL/ ，而其他平台上是 GL/。

类模板参数推导

类模板参数推导（Class template argument deduction，CTAD）允许编译器通过构造函数的参数推导模板参数。

对于用户自定义类型，可以使用推断指引（deduction guides）在合适的情况下指导编译器如何推断模板参数。
